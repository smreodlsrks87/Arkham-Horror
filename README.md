# 아컴호러식 카드게임 (개인 학습용)

아컴호러 LCG(리빙 카드 게임)를 참고해 만드는 디지털 협력 카드게임입니다.
원작 완전 복제가 아니라, **축소판 + 직접 설계한 시나리오**를 지향합니다.
HTML / JavaScript / CSS + Three.js(3D 맵)로 제작 중이며, 개인 학습 목적입니다.

> ⚠️ 카드 텍스트·이미지 저작권 = Fantasy Flight Games / 한국어판 = 코리아보드게임즈.
> 개인 학습용으로만 사용하며, 배포·상업적 사용은 하지 않습니다.

## 실행 방법

외부 데이터(cards.json)와 폰트·이미지를 불러오므로 **localhost로 열어야** 합니다.
`file://`로 직접 열면 fetch가 막혀 카드가 안 나옵니다.

```bash
# 게임 폴더에서
python -m http.server 8000
# 또는
npx serve
```

그런 다음 브라우저에서 `localhost:8000/arkham_game.html` (시작 메뉴) 또는
`localhost:8000/scenario1.html` (게임판)을 엽니다.

## 폴더 구조

```
게임폴더/
├─ arkham_game.html          # 시작 메뉴 → 덱빌더 → 프롤로그
├─ scenario1.html            # 메인 게임판 (3D 맵·손패·조우덱·토큰)
├─ cards.json                # 카드 기본정보 301장 (코어 185 + 개정코어 116)
├─ notz_player_cards.json    # 카드 "작동 방식"(효과) 데이터 — 직접 구조화
├─ card_schema_dictionary.md # 카드 효과 용어 대응표(스키마)
├─ images/                   # 토큰·카드뒷면·조사자 등 이미지
├─ fonts/                    # 아컴 아이콘 폰트 등
├─ cutscenes/                # 컷신 영상·음성·인트로
└─ (그 외 *.html)            # 제작에 쓴 편집기·도구들
```

## 데이터 설계 (핵심)

카드 정보를 **두 파일로 분리**합니다. 서로 겹치지 않습니다.

- **cards.json** = "카드가 **무엇인가**" — 이름·클래스·비용·능력치·특성·약점여부·텍스트
  (ArkhamDB에서 받은 원본. 이 파일은 안 고침)
- **notz_player_cards.json** = "카드가 **어떻게 작동하는가**" — 효과(abilities)를 코드가 실행할 구조로 정리
  (직접 만드는 부분. cards.json에 없는 정보만 담음)

효과 용어는 모두 영어로 통일했고, `card_schema_dictionary.md`에 대응표가 있습니다.

## 진행 상황

**완료**
- 카드 표시·한국어 정보, ArkhamDB 연동, 3D 맵(서재·지하실 등), 손패(부채꼴)·드래그 플레이
- 세로형 조사자 카드(CSS 크롭 + 능력치·체력/정신 오버레이), 자원·단서 토큰
- 조우덱·버린더미·혼돈주머니·파멸 UI, 페이즈/턴 루프, 멀리건(약점 제외 처리)
- 맵 조사 → 단서 연동, 컷신 시스템(인트로 크로스페이드)
- 카드 효과 데이터 스키마 확정 + 플레이어 카드 33장 작동방식 정리
- Fast/반응 카드 규칙(자기 턴 제한, when 조건) 코드 반영

**진행 중**
- 로랜드 덱 카드 완성 (수호자 16장 완료 / 탐구자·중립 진행)
- 카드 효과 실행 엔진(처리기) — 스키마의 deferred 항목들

**예정**
- 적/조우 시스템(hunter·prey·등장), 반응/타이밍 창, 판정 시스템 확장
- 시나리오 데이터(장소·조우세트·승패 분기), 세이브/로드, Electron .exe 포장

## 기술 스택

HTML / JavaScript / CSS / Three.js(r128) · 데이터는 JSON 분리 · 세이브는 localStorage 예정
